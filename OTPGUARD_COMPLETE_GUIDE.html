🔐 What is OTPGuard?

Core Purpose

OTPGuard is a dedicated OTP management platform that solves the challenge of secure authentication. Instead of businesses building their own OTP systems, they can:

  • Integrate OTPGuard's API to send OTPs to users
  • Verify OTPs securely through the platform
  • Access analytics and audit logs
  • Scale from hundreds to millions of OTPs per day

Problem OTPGuard Solves

  • Time Cost: Building OTP from scratch takes months; OTPGuard is instant
  • Security Risk: Developers implementing OTP poorly increases breach risk; OTPGuard is battle-tested
  • Operational Burden: Managing SMS providers, retry logic, and delivery is complex; OTPGuard handles it
  • Compliance: OTPGuard ensures PCI-DSS, GDPR, and local regulations compliance
  • Cost: OTPGuard's usage-based model beats building in-house for most companies

Data Flow: OTP Send & Verify

Business → OTPGuard → User → Business → OTPGuard → Success

  1. Business calls POST /api/mfa/otp/send with phone/email
  2. OTPGuard generates 6-digit code + expiry (5 min)
  3. OTP sent via SMS/Email through provider (Twilio/AWS SES)
  4. User receives code and submits back to business
  5. Business calls POST /api/mfa/otp/verify with code
  6. OTPGuard validates code against stored record
  7. Returns success/failure response
  8. OTPLog records audit trail

Core Tables:

  • users: id, email, phone, password_hash, mfa_method, mfa_enabled, plan, role, created_at
  • otp_logs: id, user_id, code, method (email/sms/totp), status (pending/verified/expired/failed), timestamp, expires_at, ip_address
  • devices: id, user_id, ip, location, user_agent, trusted, last_seen, created_at
  • api_keys: id, user_id, key, is_active, last_used, created_at

🔒 Security & Compliance

Security Best Practices Implemented

  • Password Hashing: bcrypt with salt rounds=12
  • JWT Tokens: Signed with HS256, 2-hour expiry
  • API Key Storage: Hashed, never exposed in responses
  • OTP Expiry: 5 minutes, automatically invalidated
  • Rate Limiting: 10 requests/minute per IP for auth endpoints
  • CORS: Configured to allow only trusted origins
  • Database Queries: SQLAlchemy ORM prevents SQL injection
  • Error Messages: Vague messages to prevent information leakage

Security Improvements Needed

  • HTTPS enforcement (SSL/TLS certificates)
  • CSRF tokens for form submissions
  • Content Security Policy headers
  • Two-factor authentication for admin accounts
  • IP whitelisting for admin access
  • Regular security audits (quarterly)
  • Dependency scanning for vulnerabilities
  • Encryption at rest for sensitive data

Compliance Requirements

GDPR (EU Users)

  • Privacy policy with clear data usage
  • User consent for data collection
  • Right to access (user data export)

Data Protection Policy

Data Collection

OTPGuard collects: email, phone, name, password (hashed), IP address, device info, OTP logs

Data Retention

OTP attempts: 90 days (audit trail), Device logs: 1 year, User accounts: Until deletion

Data Sharing

Never shared with third parties (except SMS providers for SMS delivery)

User Rights

Users can request data export or deletion via account settings
